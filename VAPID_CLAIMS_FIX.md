# VAPID Claims Fix - Push Notification Issue

## Problem
Notifications are failing with error: `❌ Notification Error: Missing 'sub' from claims. 'sub' is your admin email as a mailto: link.`

## Root Cause
The `VAPID_CLAIMS_SUB` environment variable is not properly configured. The application needs a valid admin email address to send Web Push notifications.

## Solution

### For Google Cloud Run Deployment

1. **Set the VAPID_CLAIMS_SUB environment variable:**
   ```
   VAPID_CLAIMS_SUB=mailto:admin@yourdomain.com
   ```
   Replace `admin@yourdomain.com` with your actual admin email.

2. **Or, set ADMIN_EMAIL as fallback:**
   ```
   ADMIN_EMAIL=admin@yourdomain.com
   ```

### Steps to Update Cloud Run Service

#### Option A: Using Cloud Console
1. Go to [Cloud Run Console](https://console.cloud.google.com/run)
2. Select your `mini-hrm` service
3. Click **EDIT & DEPLOY NEW REVISION**
4. Scroll to **RUNTIME > Set environment variables**
5. Add or update:
   - `VAPID_CLAIMS_SUB`: `mailto:admin@yourdomain.com`
   - OR `ADMIN_EMAIL`: `admin@yourdomain.com`
6. Click **DEPLOY**

#### Option B: Using gcloud CLI
```bash
gcloud run deploy mini-hrm \
  --update-env-vars VAPID_CLAIMS_SUB=mailto:admin@yourdomain.com \
  --region asia-southeast1 \
  --project clinic_hrm_system
```

### For Local Development

Add to your `.env` file:
```
VAPID_CLAIMS_SUB=mailto:admin@yourdomain.com
ADMIN_EMAIL=admin@yourdomain.com
```

## How It Works Now

The application now:
1. ✅ First checks for explicit `VAPID_CLAIMS_SUB` environment variable
2. ✅ Falls back to `ADMIN_EMAIL` if `VAPID_CLAIMS_SUB` is not set
3. ✅ Automatically formats email as `mailto:email@domain.com` if needed
4. ✅ Gracefully handles errors if no valid email is available
5. ✅ Logs any push notification failures without crashing approval/submission

## Testing

After setting the environment variable:

1. Employee submits an attendance request → Admin should receive push notification
2. Admin approves the request → Employee should receive push notification
3. Check browser notification bell for incoming notifications

## Environment Variables Reference

| Variable | Required | Format | Example |
|----------|----------|--------|---------|
| `VAPID_CLAIMS_SUB` | ❌ Optional (preferred) | `mailto:email@domain.com` | `mailto:admin@example.com` |
| `ADMIN_EMAIL` | ❌ Optional (fallback) | Email or `mailto:email@domain.com` | `admin@example.com` |
| `VAPID_PUBLIC_KEY` | ✅ Required | Base64 encoded | Generated by `npx web-push generate-vapid-keys` |
| `VAPID_PRIVATE_KEY` | ✅ Required | Base64 encoded | Generated by `npx web-push generate-vapid-keys` |

## If Still Not Working

1. Check Cloud Run logs:
   ```bash
   gcloud run logs read mini-hrm --region=asia-southeast1 --limit=100
   ```

2. Look for `VAPID_CLAIMS_SUB loaded:` line in logs - should show your email

3. Ensure the browser's notification settings allow your domain

4. Check that push subscriptions are being saved:
   - Go to database and query `push_subscription` table
   - Should have entries for logged-in users
